## 📌 Overview

Deep Sentinel is an internal threat detection system that continuously monitors user activity across your network. When suspicious behavior is detected, it automatically takes action and provides plain English explanations of why the alert was triggered — no cybersecurity expertise required.

The system monitors file access, USB devices, login attempts, emails, and network traffic. If something looks suspicious, Deep Sentinel blocks the threat, captures evidence, and provides clear explanations.

---

## ✨ Features

### 📁 File Monitoring
- Real-time tracking of file creation, modification, deletion, and renaming
- Monitors specific folders or entire drives
- Logs all file operations with timestamps and user details

### 💾 USB Device Monitoring
- Detects USB insertion and removal events
- Logs device details (manufacturer, serial number, volume name)
- Blocks unauthorized USB devices

### 🔐 Login/Logoff Tracking
- Captures successful and failed login attempts
- Tracks session activity
- Flags off-hour logins

### 📧 Email Monitoring (Outlook)
- Intercepts outgoing emails
- Scans subject, content, and attachments
- Blocks suspicious emails

### 📸 Screenshot Capture
- Captures screenshots on alerts
- Saves as forensic evidence
- Includes timestamps

### 🌐 Multi-LAN Support
- Monitors multiple network segments
- Detects unusual traffic patterns

### 🍯 Honeypot
- Deploys decoy files
- Triggers alerts on access

### 📊 Real-time Dashboard
- Live activity graphs
- Alert distribution
- User activity tracking

### ⚡ Automated Actions
- Network blocking
- Email blocking

### 📝 Explainable AI
- Converts alerts into simple English explanations
- Improves understanding for non-experts

### 📄 Report Generation
- Generates HTML/PDF reports
- Includes timeline and evidence

Recommended:

- Windows 10 or Windows 11

Some modules are optional. If a dependency is missing, the agent usually disables that feature and continues running.

## Configuration

Main server thresholds are in:

```text
config.json
FS/insepticon/config.json
```

Important settings:

```json
{
  "thresholds": {
    "files_created_today": 12,
    "http_requests_today": 500,
    "bytes_downloaded_today": 104857600,
    "large_usb_transfer_bytes": 52428800
  },
  "risk_display_threshold": 0.5
}
```

`large_usb_transfer_bytes` controls when a copied USB file becomes a critical transfer alert. The current default is 50 MB.

Agent settings are in `FS/insepticon/file_agent.py` and `config.json` generated by the agent. Important values include:

- `server_url`: server receiver URL, usually `http://127.0.0.1:5000/receive_log`
- `watch_paths`: folders monitored for file activity
- `max_upload_risk_size_mb`: large-file threshold used by the agent
- `enable_process_monitor`: process monitoring switch
- `enable_outlook_monitor`: Outlook monitoring switch
- `enable_imap_monitor`: IMAP monitoring switch

## Running Locally


Start the server:

```bash
python FS/insepticon/server.py
```

Open the dashboard:

```text
http://127.0.0.1:5000/dashboard
```

Start the endpoint agent in another terminal:

```bash
python FS/insepticon/file_agent.py
```

For logon/logoff monitoring, run the agent terminal as Administrator. Without admin rights, Windows Security Event Log monitoring may be skipped.

## Detection Coverage

| Detection area | Event type | Alert behavior |
| --- | --- | --- |
| After-hours login | `logon` / `user_logon` | Creates `after_hours_login` alert |
| After-hours logoff | `logon` / `user_logoff` | Creates `after_hours_logoff` alert |
| USB plug-in | `usb` / `plugged` or `device_connected` | Logged and risk-scored |
| USB removal | `usb` / `unplugged` or `device_removed` | Logged |
| Large USB transfer | `usb` / `large_file_transfer` | Creates critical `large_usb_transfer` alert |
| File activity | `file` | Created, modified, deleted, moved events |
| Sensitive keywords | `file` | Raises file risk score |
| Process activity | `process` | Suspicious process start/end events |
| Clipboard activity | `clipboard` | Sensitive copied text detection |
| Email activity | `outlook`, `imap`, `email` | ML classification and approval queue |
| Honeypot access | honeypot events | Critical incident and response workflow |

## Security Notice

DeepSentinel is a monitoring and response prototype. Use it only on systems where you have permission to monitor activity. Logon/logoff monitoring may require Administrator access because it reads Windows Security Event Logs. Some response actions, such as blocking or isolation, can affect normal user activity and should be tested in a controlled environment first.
